Existing Workflow Analysis
海外学生通过中介或自主向Origin Institute提交入学申请材料。Admin staff登录MeshedRTO查看申请材料，并根据一套评定矩阵对学生进行风险评估，评估维度包括国籍、拒签记录、庇护签证申请历史、年龄、签证过期记录等。根据评估结果，staff决定直接拒绝、发出full offer或发出conditional offer。
学生收到offer后，可能手持多所学校的offer进行比较。如学生在规定时间内无回应，staff需要手动跟进follow up。若学生签署offer，则需在规定时间内补全所有申请材料。
材料确认无误后，学生需缴纳initial payment，onshore学生为1500澳元，offshore学生为2500澳元。部分学生在此阶段选择缴纳超过50%的学费，超出部分作为额外deposit处理。Finance staff收到付款后，需分别在Xero和MeshedRTO中手动录入付款记录，因两个系统相互独立。
付款确认后，admin staff登录政府系统PRISMS，手动录入学生信息并签发COE（Confirmation of Enrolment）。PRISMS与MeshedRTO同样互不关联，信息无法自动同步。
Pain Point Identification
学生质量评定依赖人工逐条判断 目前每个申请进来，staff需要人工逐一核对评定矩阵中的各项风险指标，效率低，且不同staff的判断标准可能存在差异，影响评定一致性。
Offer跟进缺乏自动化机制 学生签署offer后若无回应，目前需要staff人工记得跟进，没有系统提醒，容易漏跟，导致潜在学生流失。
Payment信息重复录入两个系统 Finance staff收到付款后需分别在Xero和MeshedRTO各录入一次，同一条数据做两遍，耗时且存在两个系统数据不一致的风险。
各节点通知邮件全靠人工 从收到申请、发offer、payment确认到COE签发，每个节点的学生通知邮件均需staff手动，措辞可统一，但耗时且存在遗漏风险。
Automation Opportunities
机会一：学生风险评定自动评分 根据现有评定矩阵，搭建一个自动评分系统。学生提交申请后，系统自动对照各项风险指标生成风险等级：高风险自动标红并提示建议拒绝，低风险标绿建议进入offer流程，中间地带推送人工审核。Staff只需重点处理中间地带的申请，大幅减少逐条人工判断的工作量。
机会二：全流程节点通知邮件自动触发 针对招生流程中每个关键节点，预设标准化邮件模板，由Power Automate根据状态变化自动触发发送，包括：申请收到确认、conditional/full offer发送、材料补全提醒、payment确认、COE签发通知。所有邮件措辞统一，无需staff逐一撰写
机会三：Offer签署后自动跟进 学生收到offer后若在设定时间内无回应，Power Automate自动触发follow up邮件，可设置多轮提醒，超过最终期限自动标记为无效申请并通知staff，无需人工记忆跟催。
机会四：材料补全自动追踪与提醒 Conditional offer学生需补全材料，系统自动记录材料清单和截止日期，材料未到期前自动提醒学生，截止日期临近再次提醒，逾期自动通知staff介入处理
机会五：Payment录入减少重复操作 如MeshedRTO与Xero的官方集成开通，payment信息在一个系统录入后自动同步至另一个系统，消除重复录入。短期内如集成未开通，可用Power Automate搭建中间层，将MeshedRTO导出数据格式化为Xero可导入的格式，减少手动录入工作量。
机会六：COE签发前信息自动整理 PRISMS作为政府系统无法直接集成，但可以在COE签发前自动从MeshedRTO提取学生关键信息，生成一张标准化的录入清单，staff直接对照清单在PRISMS录入，减少翻找信息的时间，同时降低录入出错的概率。
Feasibility

机会一：学生风险评定自动评分 — 中等可行性 前提是评定矩阵的标准已经明确量化，每项指标有清晰的判断规则。标准明确后，可用Power Automate或简单的Excel逻辑实现自动评分，无需额外付费。若矩阵标准尚未完全量化，需先由管理层将评定标准文档化，再进行自动化落地。

机会二：全流程通知邮件自动触发 — 高可行性 Power Automate配合Outlook标准模板完全可以实现，在Microsoft 365套餐范围内无额外成本。

机会三：Offer签署后自动跟进 — 高可行性 Power Automate的定时触发和条件判断完全支持此场景，无额外成本，落地周期约3至5天，能直接减少潜在学生流失。

机会四：材料补全自动追踪与提醒 — 高可行性 逻辑与机会三类似，Power Automate原生支持，无额外成本，与机会二和三可以同步搭建，复用相同的流程框架。

Existing Workflow Analysis
Agent Commission：处理现状，每月15日，staff手动登录MeshedRTO，进入Bulk Agent Payment模块，逐个点开每个agent，下载各自的学生commission Excel表。下载完后人工检查每个表里的学生状态，确认只有Current Students才纳入返佣。然后打开Outlook，一封一封写邮件，手动附上对应agent的Excel，逐个发送（不能群发，隐私原因）。Agent收到后根据表格数据开具invoice回传。Staff再人工核对invoice金额与表格是否一致。核对无误后手动保存到OneDrive，再登录Xero，手动逐条录入invoice信息，最后完成放款。 

Agent management 
new agent assessment：新中介主动联系Origin Institute表达合作意向，staff通过邮件或其他渠道收取材料，材料包括business profile、联系人信息、公司注册号、MIRA/PIER执照等。由于没有标准化提交渠道，材料的格式、种类、完整性参差不齐，staff需要人工逐一检查是否齐全，缺材料还需来回追。材料通过后进行reference check，联系agent现有合作学校确认合作质量，通过后在Adobe上签订合作协议，最后在MeshedRTO里创建agent账号并激活。 
current agent review：基于现有agent推荐的学生数量、质量、学生表现等多维度指标，staff人工维护一张Excel表（即Agent Status Check表），定期对agent进行评级和状态判断。评定涉及数据分析、政策考量、主观印象和未来合作发展判断，目前完全也只能依赖人工。 
renew or terminate a contract ：合同到期前需要staff主动发现并跟进。Renew流程需要人工起草新合同，发给agent签署（Adobe），然后回到MeshedRTO手动更新合同信息并激活。Terminate流程则要求在7天内向政府（ASQA）申报，目前靠人工定期查阅和跟进，没有系统提醒。
Pain Point Identification
Agent Commission： 
重复劳动多： 下载、拆分、发邮件、录入Xero，每一步都是纯手工重复操作，随agent数量线性增长。十几个到未来更多可能？ 
容易出错： 人工检查学生状态容易漏掉canceled学生，导致多付佣金。invoice核对靠肉眼比对，金额或姓名错误不易发现。 
信息孤岛： MeshedRTO和Xero数据不互通，同一条数据要录入两个系统，产生双重工作量和数据不一致风险。（成本与限制） 
发送风险： 逐个发邮件，一旦发错附件给错agent，涉及学生隐私和财务纠纷，风险较高。 

 Agent management： 材料收集标准化：MeshedRTO里已有agent注册入口，但部分字段未设为必填，导致材料提交不完整。此外onshore和offshore agent所需材料不同（难量化判断） 合同到期缺乏提醒：从Agent Status Check表可以看到，Expiry Date格式不统一，部分为文字，部分为日期格式，说明现在完全依赖人工记忆或定期翻表，容易静默过期 terminate 用人记忆去做：7天内ASQA申报是合规要求，但目前没有任何系统触发提醒，完全依靠staff自觉，一旦遗漏面临合规风险。

机会四：MeshedRTO与Xero数据同步（X 不可行 需要更多的订阅费用） Meshed官方已有Xero集成功能，可以自动将invoice数据推送到Xero，消除手动录入环节。需确认是否已开通。

Existing Workflow Analysis
Transfer to another school: 学生决定转出
1.确认该学生的COE是否已超过6个月且正在就读principal course。资格确认后，学生填写纸质或电子申请表，并根据转出原因提交对应的supporting evidence
2.转出原因大体分三类：Misconduct 严重违纪、不可抗力 如自然灾害、 健康等私人因素（需要medical certificate）除evidence外，学生还需同时提交接收学校的offer letter，证明已有明确的去向。
3.staff人工审阅所有文件，判断材料是否齐全、理由是否充分，最终给出三种结果之一：Approve、Refuse、或Need More Evidence。结果确定后，staff手动撰写通知邮件发送给学生，说明结果及后续步骤。
Pain Point Identification
资格核查依赖人工 每个申请staff都需要手动登录MeshedRTO查询学生的COE开始日期和当前就读课程，判断是否满足转出资格。这一步重复性高，且容易因疏忽出错
材料要求不清晰 学生在申请前往往不清楚自己需要提交哪些材料，不同原因对应不同evidence，如果没有明确指引，容易提交不完整，导致来回追料，拖慢处理周期
结果通知依赖人工撰写 三种结果对应三种不同的通知内容，目前每次都需要staff从头写邮件或套用模板，效率低，措辞也可能不一致
材料完整性没有前置检查 材料是否齐全目前完全靠staff第一轮人工筛查，缺材料才退回，学生重新补交再进入队列，无谓延长整个处理周期。
Automation Opportunities
1.资格自动校验 学生提交申请时，系统自动调取MeshedRTO数据核查COE是否超过6个月、当前是否就读principal course。不符合条件直接自动退回并说明原因，符合条件才允许进入申请流程，完全消除人工查询这一步
2.动态材料清单 申请表根据学生选择的转出原因，自动显示对应的evidence要求。选择健康因素，自动列出需要医疗证明；选择不可抗力，自动列出对应文件要求。同时强制要求上传offer letter，三类材料缺一项无法提交
3.材料完整性自动检查 提交后系统自动检查所有required文件是否已上传，缺材料直接触发Need More Evidence状态并自动通知学生注明缺少什么，无需staff进行第一轮人工筛查
4.结果通知自动发送 Staff在系统中点击Approve、Refuse或Need More Evidence后，自动触发对应的标准化邮件模板发送给学生，邮件内容包含结果说明和后续步骤，措辞统一规范，无需人工撰写。
Feasibility
机会一：资格自动校验 — 中等可行性 技术上可行，但依赖MeshedRTO的API接口能否实时返回学生COE和就读课程数据。如API已开通则可直接实现，如未开通则需评估开通成本。这是整个流程自动化的关键前提
机会二：动态材料清单 — 高可行性 用Microsoft Forms或MeshedRTO内置表单均可实现条件显示逻辑，强制上传字段配置简单，无需额外付费
机会三：材料完整性自动检查 — 高可行性 Power Automate可以在表单提交后自动检查附件是否完整，触发对应的通知邮件
机会四：结果通知自动发送 — 高可行性 Power Automate配合Outlook标准模板完全可以实现
